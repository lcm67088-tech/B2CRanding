# Codex 작업 규칙

## 프로젝트 기준

- 이 저장소는 `landing.papainsight.co.kr` 랜딩 페이지의 정적 파일을 관리한다.
- `GitHub`(깃허브: 코드 저장소 서비스) 저장소는 `https://github.com/lcm67088-tech/B2CRanding.git`이다.
- 실제 화면 기본 파일은 `index.html`, `styles.css`, `script.js`이다.
- 이미지와 기타 정적 리소스는 `assets/`를 기준으로 사용한다.

## 작업 원칙

- 기존 사용자가 만든 변경사항은 임의로 되돌리지 않는다.
- `HTML`(에이치티엠엘: 화면 구조 코드), `CSS`(씨에스에스: 화면 스타일 코드), `JS`(제이에스: 화면 동작 코드)를 바꿀 때는 변경 전 파일을 `archives/` 아래에 보관한다.
- 변경 후에는 브라우저에서 데스크톱, 태블릿, 모바일 화면을 확인한다.
- 작업 내용은 `PROJECT_NOTES.md`, `CHANGELOG.md`, `README.md`에 필요한 범위만 최신화한다.
- 실패한 명령이나 잘못된 가정은 `ERROR_LOG.md`에 남긴다.
